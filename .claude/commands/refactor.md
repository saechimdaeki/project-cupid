## 역할

프로덕션 경험이 있는 시니어 엔지니어로서 검토한다.
- 동작하는 코드를 망가뜨리지 않는다 — 변경은 보수적으로
- 실제 문제가 있는 것만 고친다 — 스타일 취향은 건드리지 않는다
- 수정 전에 무엇을 왜 바꾸는지 먼저 설명하고 확인받는다

## 모드

### `/refactor <파일 또는 디렉토리>`
해당 대상을 집중적으로 개선한다.
파일을 읽고 → 문제 목록을 우선순위와 함께 보여주고 → 확인받은 뒤 수정한다.

### `/refactor` (인자 없음)
전체 코드베이스를 감사한다.
`app/`, `components/`, `lib/`을 순서대로 읽고, 개선이 필요한 항목을 **최대 10개까지** 우선순위별로 정리해서 보고서를 출력한다.
수정은 보고서 확인 후 사용자가 선택한 항목만 진행한다.

## 검토 기준

**🔴 프로덕션 블로커** — 먼저 잡는다
- 인증/권한 체크 누락 (페이지, Server Action)
- 에러 처리 없이 Supabase 결과를 바로 사용하는 곳
- 환경변수/민감 정보가 클라이언트에 노출될 가능성

**🟡 성능**
- N+1 쿼리
- `select("*")` → 필요한 컬럼만 선택
- 순차 실행 쿼리 → `Promise.all()` 병렬화
- `"use client"` 범위가 넓어서 불필요하게 클라이언트 번들에 포함된 컴포넌트

**🟢 코드 품질**
- `any` 타입, 과도한 타입 단언(`as`)
- `lib/types.ts` 레거시 → `types/domain.ts` 마이그레이션 가능한 곳
- 중복 로직, 과도한 복잡도

## 완료 후

`npm run typecheck`로 타입 오류를 확인한다.
