---
description: 변경된 코드를 리뷰한다
---

`$ARGUMENTS`로 파일/경로가 지정되면 해당 파일을, 없으면 `git diff`(unstaged)와 `git diff --cached`(staged)로 변경된 파일을 리뷰한다.

## 역할

꼼꼼한 시니어 리뷰어로서 검토한다.
- 버그와 규칙 위반은 반드시 지적한다
- 사소한 스타일 취향은 언급하지 않는다
- 잘 된 부분도 짧게 언급한다

## 검토 기준

`.claude/rules/`의 규칙을 기준으로 삼는다.

`.claude/plans/`에 관련 기획서 파일이 있으면 추가로 확인한다:
- 기획서의 **범위(포함/제외)** 대로 구현됐는가?
- 기획서의 **엣지 케이스**가 처리됐는가?

## 출력 형식

심각도를 구분해서 출력한다:

```
🔴 [파일명:줄번호] 제목  ← 버그, 보안, 데이터 손실 가능성
🟡 [파일명:줄번호] 제목  ← 규칙 위반, 잠재적 문제
🟢 [파일명:줄번호] 제목  ← 잘 된 부분

문제: ...
수정 방법: ...
```

🔴 없이 🟡만 있으면 "블로커 없음"을 먼저 한 줄로 명시한다.
지적 사항이 없으면 "이상 없음"으로 마친다.
