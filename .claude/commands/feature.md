`$ARGUMENTS`에 명시된 기능을 구현한다.
인자는 기획서 파일명(`candidate-filter`) 또는 기능 설명 문장(`후보 필터링 기능`) 모두 가능하다.

## 구현 전 확인

`.claude/plans/` 에서 관련 기획서 파일을 찾아 읽는다.
- 기획서의 **범위 결정(포함/제외)** 을 반드시 지킨다 — 기획에 없는 기능을 추가하지 않는다
- 기획서의 **엣지 케이스** 를 구현에 반영한다
- 기획서의 **구현 범위** 체크리스트를 따라 순서대로 진행한다

기획서가 없으면 사용자에게 `/plan`으로 기획서를 먼저 만들지 확인한다.
사용자가 그대로 진행하겠다고 하면 관련 기존 파일을 읽어 패턴을 파악한 뒤 **최소 범위로** 구현한다.
항상 `types/domain.ts`, `lib/types.ts`를 확인하고, 비슷한 기존 기능이 있으면 그 패턴을 따른다.

## 구현

기획서의 구현 범위 체크리스트 순서대로 진행한다.
`.claude/rules/`의 규칙을 준수한다.

## 구현 완료 후

`npm run typecheck`로 타입 오류를 확인한다.
