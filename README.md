## 왜 이게 필요한가요?

소개팅 주선은 생각보다 복잡합니다.

- 누구와 누구를 연결했는지 기억이 흐릿해집니다.
- 한 번 잘 안 된 사이를 모르고 다시 연결할 수 있습니다.
- 여러 사람의 상태를 동시에 머릿속에서 관리하다 보면 타이밍을 놓칩니다.
- 엑셀이나 메모앱으로는 흐름이 보이지 않습니다.

**Project Cupid는 이 모든 문제를 해결하기 위해 만들었습니다.**

단순히 "사람을 등록하는 리스트"가 아니라, 누가 지금 어떤 상태인지, 어떤 흐름이 이어지고 있는지, 왜 안 됐는지를 한눈에 파악하고 기록으로 남기는 **매칭 운영 보드**입니다.

---

## 핵심 기능

### 후보 관리 — 조건이 아니라 사람을 봅니다

<img width="1453" height="732" alt="후보 관리 화면" src="https://github.com/user-attachments/assets/1e19f11c-66e5-41e1-99b0-ecc60c9dca14" />

- 이름, 나이, 직업, 지역, MBTI, 외모 인상, 비공개 메모까지 한 카드에 정리
- 사진 갤러리로 실물 인상을 바로 파악
- "적극검토 → 매칭진행중 → 커플완성" 상태를 드래그 없이 직관적으로 이동
- 필터·검색으로 수십 명의 후보 중 딱 맞는 사람을 즉시 찾기

---

### 플로우 보드 — 전체 상황을 한눈에

- 현재 적극 검토 중인 사람, 매칭이 진행 중인 커플, 이미 성사된 커플을 컬럼별로 정렬
- 지금 당장 연결 가능한 후보가 몇 명인지 숫자로 파악
- 대시보드에서 바로 매칭 페어를 연결하고 상태를 올릴 수 있음

---

### 매칭 타임라인 — 기록이 곧 판단력입니다

<img width="1495" height="752" alt="타임라인 화면" src="https://github.com/user-attachments/assets/45f890d1-2e86-4e00-b73b-c5825a5fa769" />

- 소개 시작 → 첫 만남 → 후속 진행 → 커플완성 / 종료 흐름을 타임라인으로 기록
- 언제 무슨 일이 있었는지 날짜와 함께 남겨 다음 판단의 감각을 쌓음
- 최근 매칭 기록을 대시보드 오른편에서 실시간으로 확인

---

### Match Detail 모달 — 두 사람을 나란히 봅니다

<img width="1140" height="655" alt="Match Detail 모달" src="https://github.com/user-attachments/assets/0464cd42-feb6-4fb3-a029-ae5bd951cca6" />

- 타임라인 카드를 누르면 매칭된 두 사람의 프로필 사진과 정보가 나란히 펼쳐짐
- 나이, 직업, 지역, MBTI, 종교 등 핵심 정보를 칩 형태로 한 번에 비교
- 어떤 단계에서 무슨 이유로 진행되고 있는지 요약 메모 포함

---

### 과거 이력 관리 — 주선자의 실수를 원천 차단합니다

<img width="1026" height="722" alt="과거 이력 화면" src="https://github.com/user-attachments/assets/b6ee97e4-7778-43ce-b4df-8bbfd4738f67" />

- 한 번 안 된 사이를 모르고 다시 연결하는 실수를 원천적으로 방지
- 매칭 종료 시 "왜 안 됐는지" 사유를 반드시 기록
- 상대 프로필 상세 페이지에서 과거 이력을 바로 확인 가능
- 비공개 메모로 주선자만 볼 수 있는 맥락 정보를 보관

---

### 운영 데스크 — 프로필 페이지에서 모든 걸 처리합니다

- 후보 상태 변경, 매칭 종료, 커플 확정까지 한 페이지에서 처리
- 커플이 성사되면 칸반 흐름이 즉시 "커플완성" 컬럼으로 반영
- 비공개 메모, 매칭 MATCH FLOW(진행 중 / 커플완성 / 종료) 칸반을 한 화면에서 확인

---

## 이런 상황에 씁니다

| 상황 | Project Cupid로 해결 |
|------|----------------------|
| 누구를 다음에 소개할지 막막할 때 | 플로우 보드에서 적극검토 중인 후보를 바로 파악 |
| A와 B가 예전에 만난 적 있는지 기억 안 날 때 | 과거 이력에서 종료 사유까지 바로 확인 |
| 매칭 진행 상황을 동료와 공유할 때 | 승인된 운영자 전원이 같은 보드를 실시간으로 봄 |
| 커플이 성사됐을 때 기록을 남기고 싶을 때 | 운영 데스크에서 커플 확정 → 타임라인 자동 기록 |
